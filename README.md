# SecPay - Utility Billing Platform

SecPay is a comprehensive Full-Stack SaaS application demonstrating cloud-ready architectural patterns, secure payment logic, and decoupled frontend/backend services.

Developed across 7 structured phases, this project exhibits professional Backend Engineering standards utilizing Python, FastAPI, SQLAlchemy, and MySQL, supported by a lightweight, Vanilla JavaScript frontend.

## Architecture

The system utilizes a strictly layered REST API Architecture:
- `backend/app/routes/`: Transport layer mapping HTTP requests.
- `backend/app/schemas/`: Data Transfer Objects (DTOs) constructed with Pydantic for rigid input/output validation.
- `backend/app/services/`: Core decoupled business logic for CRUD operations.
- `backend/app/models/`: Database tables and relational constraints mapped via SQLAlchemy ORM.
- `backend/app/utils/`: Security operations isolating password hashing (`bcrypt`) and token manufacturing (`JWT`).

## Getting Started

### 1. Database Setup
Ensure you have MySQL installed and running locally on port `3306`.
Create the `secpay_db` database:
```sql
CREATE DATABASE secpay_db;
```

### 2. Backend Initialization
1. Navigate to the backend directory: `cd backend`
2. Create and activate a Virtual Environment if you haven't already:
   - Windows: `python -m venv venv` and `.\venv\Scripts\activate`
   - Mac/Linux: `python3 -m venv venv` and `source venv/bin/activate`
3. Install the API requirements:
   - `pip install -r requirements.txt`
4. Setup your `.env` variables containing your DB credentials (already instantiated for local ops).
5. Start the underlying FastAPI server utilizing `uvicorn`:
   - `uvicorn main:app --reload`
6. View the autogenerated OpenAPI Swagger documents at `http://localhost:8000/docs`.

### 3. Frontend Initialization
The frontend is completely detached and relies entirely on native browser APIs.
1. Serve the frontend directory using any static tool. For example:
   - `cd frontend`
   - `python -m http.server 3000`
2. Open your web browser and visit `http://localhost:3000/index.html`.

## Security Implementations
- **Stateless Authentication**: Powered by securely signed JSON Web Tokens (JWT) protecting API endpoints natively.
- **Password Obfuscation**: Passwords are mathematically altered utilizing salt and `bcrypt` slow-hashing algorithms before persistent storage.
- **SQL Injection Prevention**: Pydantic schemas filter malicious strings, and SQLAlchemy automatically escapes all incoming variables safely constructing parameterized database requests.
- **Vertical Authorization Rules**: Logged-in credentials strictly guarantee users can never view or manipulate bills/payments tethered maliciously to another user ID.
